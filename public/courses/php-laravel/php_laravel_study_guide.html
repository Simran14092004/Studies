Days 1–4 · PHP Basics
PHP Fundamentals

PHP is a server-side scripting language. Every .php file runs on the server and sends HTML to the browser. Master these before anything else.

Variables, Data Types & Echo Code

PHP variables start with $. PHP is loosely typed — you don't declare types. The echo or print statement outputs to the browser.

variables_types.php
<?php
// Variables always start with $
$name    = "Riya";          // string
$age     = 22;              // integer
$salary  = 45000.50;        // float
$isIntern = true;           // boolean
$company = null;            // null

// Outputting values
echo "Hello, " . $name;     // concatenation with .
echo "<br>Age: {$age}";     // string interpolation

// Type checking
var_dump($isIntern);         // bool(true)
echo gettype($salary);      // double

// Type casting
$strAge = (string) $age;
$intStr = (int) "42abc";    // 42
Real-World Use at 24/7 Software Inc.

When a user fills a registration form, their name, email, and age come in as variables: $name = $_POST['name']. You'll validate, cast, and store these in a database every single day.

Conditionals & Loops Code

Control flow with if/else, switch, and loops. PHP has for, while, foreach — you'll use foreach constantly for arrays.

control_flow.php
<?php
$score = 85;

// if / elseif / else
if ($score >= 90) {
    echo "Grade: A";
} elseif ($score >= 75) {
    echo "Grade: B";   // this runs
} else {
    echo "Grade: C";
}

// Ternary (short if)
$status = $score >= 50 ? "Pass" : "Fail";

// for loop
for ($i = 1; $i <= 5; $i++) {
    echo "Row {$i} <br>";
}

// foreach (most common in PHP)
$fruits = ["apple", "mango", "banana"];
foreach ($fruits as $fruit) {
    echo $fruit . " <br>";
}

// while loop
$n = 1;
while ($n <= 3) {
    echo $n++;
}
Real-World: E-commerce Order List

Fetching all orders from a database returns an array. You loop through with foreach ($orders as $order) and display each row in an HTML table — this is the backbone of every dashboard.

Arrays (Indexed, Associative, Multidimensional) Code

Arrays are the workhorse of PHP. Associative arrays (key → value) are like Python dicts and JavaScript objects. You'll use them everywhere — database rows, config, API responses.

arrays.php
<?php
// Indexed array
$skills = ["PHP", "Laravel", "MySQL"];
echo $skills[0];              // PHP

// Associative array (key => value)
$user = [
    "name"  => "Arjun",
    "email" => "arjun@247.in",
    "role"  => "intern"
];
echo $user["name"];           // Arjun

// Multidimensional (array of arrays)
$employees = [
    ["id" => 1, "name" => "Priya"],
    ["id" => 2, "name" => "Rahul"],
];
echo $employees[0]["name"]; // Priya

// Useful array functions
array_push($skills, "Git");  // add to end
count($skills);              // 4
in_array("PHP", $skills);   // true
array_keys($user);          // [name,email,role]
array_merge($skills, ["Vue"]);
Real-World: API Response Handling

Every JSON API response becomes a PHP associative array after json_decode($response, true). At 24/7 Software, AIML model results come back as arrays you'll parse and display.

Functions & Built-in String Functions Code

Functions are reusable code blocks. PHP has 1000+ built-in functions — learn the string and math ones first. Use type hints (PHP 7+) for cleaner code.

functions.php
<?php
// Basic function
function greet(string $name): string {
    return "Hello, {$name}!";
}
echo greet("Anjali");

// Default parameter value
function makeTitle($text, $tag = "h1"): string {
    return "<{$tag}>{$text}</{$tag}>";
}

// Arrow function (PHP 7.4+)
$double = fn($n) => $n * 2;
echo $double(5);              // 10

// Essential string functions

substr("Laravel", 0, 3);     // "Lar"
strpos("Hello PHP", "PHP");  // 6
sprintf("Price: ₹%.2f", 99.9);

Object-Oriented PHP — Classes & Objects Theory + Code

OOP organizes code into classes (blueprints) and objects (instances). Laravel is 100% OOP — you cannot use Laravel without understanding this. Key concepts: properties, methods, constructor, visibility (public/private/protected).

oop_basics.php
<?php
class Employee {
    // Properties
    public string $name;
    private float $salary;
    protected string $department;

    // Constructor — runs when object is created
    public function __construct($name, $salary, $dept) {
        $this->name       = $name;
        $this->salary     = $salary;
        $this->department = $dept;
    }

    // Method
    public function introduce(): string {
        return "{$this->name} works in {$this->department}";
    }

    // Getter for private property
    public function getSalary(): float {
        return $this->salary;
    }
}

// Create an object
$emp = new Employee("Neha", 55000, "AIML");
echo $emp->introduce();   // Neha works in AIML
echo $emp->getSalary();  // 55000

// Inheritance
class Intern extends Employee {
    public int $daysLeft;

    public function __construct($name, $dept, $days) {
        parent::__construct($name, 0, $dept);
        $this->daysLeft = $days;
    }

    public function introduce(): string {
        return "Intern {$this->name}, {$this->daysLeft} days to go!";
    }
}

$me = new Intern("You", "AIML", 20);
echo $me->introduce();
Real-World: Every Laravel Model is a Class

In Laravel, class User extends Model is OOP inheritance. Every controller, middleware, and service is a class. Understanding $this, constructors, and visibility is non-negotiable.

Forms, GET/POST & Input Validation Code

Forms send data via GET (URL) or POST (body). Always validate and sanitize before processing. Never trust raw user input.

form_handler.php
<!-- HTML Form -->
<form method="POST" action="process.php">
    <input type="text" name="username">
    <input type="email" name="email">
    <button type="submit">Register</button>
</form>

<?php // process.php
if ($_SERVER["REQUEST_METHOD"] === "POST") {

    // Sanitize inputs
    $username = trim(htmlspecialchars($_POST["username"]));
    $email    = filter_var($_POST["email"], FILTER_SANITIZE_EMAIL);

    // Validate
    $errors = [];
    if (empty($username)) {
        $errors[] = "Username required";
    }
    if (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
        $errors[] = "Invalid email";
    }

    if (empty($errors)) {
        echo "Welcome, {$username}!";
    } else {
        foreach ($errors as $e) echo $e . "<br>";
    }
}
Security at 24/7 Software Inc.

OWASP Top 10 attacks include XSS (cross-site scripting) and SQL injection — both exploited via form inputs. htmlspecialchars() prevents XSS. Use PDO prepared statements (next topic) to prevent SQL injection.

MySQL Database with PDO Critical

PDO (PHP Data Objects) is the modern, secure way to talk to databases. Always use prepared statements to prevent SQL injection. Master CRUD — Create, Read, Update, Delete.

database_pdo.php
<?php
// 1. Connect to MySQL
try {
    $pdo = new PDO(
        "mysql:host=localhost;dbname=company_db;charset=utf8",
        "root", "password",
        [PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION]
    );
} catch (PDOException $e) {
    die("Connection failed: " . $e->getMessage());
}

// 2. CREATE — Insert (always use prepared statements)
$stmt = $pdo->prepare(
    "INSERT INTO users (name, email) VALUES (:name, :email)"
);
$stmt->execute([':name' => "Kiran", ':email' => "k@247.in"]);

// 3. READ — Fetch all users
$stmt = $pdo->query("SELECT * FROM users");
$users = $stmt->fetchAll(PDO::FETCH_ASSOC);
foreach ($users as $user) {
    echo $user['name'] . " - " . $user['email'];
}

// 4. UPDATE
$stmt = $pdo->prepare(
    "UPDATE users SET email = :email WHERE id = :id"
);
$stmt->execute([':email' => "new@247.in", ':id' => 1]);

// 5. DELETE
$stmt = $pdo->prepare("DELETE FROM users WHERE id = :id");
$stmt->execute([':id' => 3]);

Sessions, Cookies & Authentication Flow Code

HTTP is stateless — sessions solve this by storing user data server-side between requests. This is how "stay logged in" works. Cookies store small data client-side.

session_auth.php
<?php
session_start();  // Always first line

// Login logic
function loginUser($pdo, $email, $password): bool {
    $stmt = $pdo->prepare(
        "SELECT * FROM users WHERE email = :email"
    );
    $stmt->execute([':email' => $email]);
    $user = $stmt->fetch();

    if ($user && password_verify($password, $user['password'])) {
        // Store in session
        $_SESSION['user_id']   = $user['id'];
        $_SESSION['user_name'] = $user['name'];
        return true;
    }
    return false;
}

// Hash password on registration
$hashed = password_hash("mypassword", PASSWORD_BCRYPT);

// Protect a page
if (!isset($_SESSION['user_id'])) {
    header("Location: login.php");
    exit;
}

echo "Welcome, " . $_SESSION['user_name'];

// Logout
session_destroy();
header("Location: login.php");
Always use password_hash() + password_verify(). Never store plain text passwords. Laravel handles all of this automatically with its Auth system.

Interfaces, Abstract Classes & Traits Theory + Code

Interfaces define a contract (what methods must exist). Abstract classes provide partial implementation. Traits are reusable code mixins. Laravel uses all three heavily.

interfaces_traits.php
<?php
// Interface — defines contract
interface Notifiable {
    public function sendNotification(string $message): void;
}

// Implementing the interface
class EmailNotifier implements Notifiable {
    public function sendNotification(string $msg): void {
        echo "Email: {$msg}";
    }
}

class SMSNotifier implements Notifiable {
    public function sendNotification(string $msg): void {
        echo "SMS: {$msg}";
    }
}

// Trait — reusable code mixin
trait Timestampable {
    public function getCreatedAt(): string {
        return date('Y-m-d H:i:s');
    }
}

class Post {
    use Timestampable;  // mixes in the trait
    public string $title;
}

$post = new Post();
echo $post->getCreatedAt();
Laravel Connection

Laravel's Authenticatable is an interface. SoftDeletes, HasApiTokens are traits you add to your User model with use. Understanding traits is how you'll extend Laravel models.

REST APIs — Consuming & Building Critical for AIML

APIs are how modern services communicate. As an AIML intern, you'll call ML model APIs constantly. PHP uses cURL or file_get_contents to consume APIs, and JSON to exchange data.

api_client.php
<?php
// Consuming an external API (e.g. ML model endpoint)
function callMLApi(string $text): array {
    $ch = curl_init();
    curl_setopt_array($ch, [
        CURLOPT_URL           => "https://api.247ml.in/predict",
        CURLOPT_RETURNTRANSFER => true,
        CURLOPT_POST          => true,
        CURLOPT_POSTFIELDS    => json_encode(['text' => $text]),
        CURLOPT_HTTPHEADER    => [
            'Content-Type: application/json',
            'Authorization: Bearer YOUR_TOKEN'
        ]
    ]);
    $response = curl_exec($ch);
    curl_close($ch);
    return json_decode($response, true);
}

$result = callMLApi("Classify this customer review");
echo $result['sentiment'];  // "positive"

// Building a simple REST API endpoint
header('Content-Type: application/json');
$data = ['status' => 'success', 'data' => ['id'=>1]];
echo json_encode($data);

Eloquent ORM — Database the Laravel Way Must Learn

Eloquent is Laravel's ORM. Instead of writing raw SQL, you use elegant PHP methods. Each database table has a corresponding Model class. Relationships (hasMany, belongsTo) make joins effortless.

app/Models/Task.php
<?php
namespace App\Models;
use Illuminate\Database\Eloquent\Model;

class Task extends Model {
    protected $fillable = ['title', 'status', 'user_id'];

    // A task belongs to a user
    public function user() {
        return $this->belongsTo(User::class);
    }
}
Eloquent queries — no raw SQL needed!
// CREATE
Task::create(['title' => 'Learn Laravel', 'status' => 'pending']);

// READ
$tasks    = Task::all();
$task     = Task::find(1);
$pending  = Task::where('status', 'pending')->get();
$recent   = Task::latest()->take(5)->get();

// UPDATE
Task::find(1)->update(['status' => 'done']);

// DELETE
Task::find(1)->delete();

// Eager loading (prevent N+1 query problem)
$tasks = Task::with('user')->get();
foreach ($tasks as $task) {
    echo $task->title . " by " . $task->user->name;
}

Controllers, Validation & Auth Critical

Controllers handle request logic. Laravel's built-in validation is powerful. Laravel Breeze/Sanctum gives you complete auth scaffolding in minutes.

app/Http/Controllers/TaskController.php
<?php
namespace App\Http\Controllers;
use App\Models\Task;
use Illuminate\Http\Request;

class TaskController extends Controller {

    public function index() {
        $tasks = Task::latest()->paginate(10);
        return view('tasks.index', compact('tasks'));
    }

    public function store(Request $request) {
        // Validation (auto redirects back on failure)
        $validated = $request->validate([
            'title'  => 'required|string|max:255',
            'status' => 'required|in:pending,done',
        ]);

        Task::create($validated);

        return redirect()->route('tasks.index')
                         ->with('success', 'Task created!');
    }
}

# Install Auth scaffolding (one command!)
composer require laravel/breeze --dev
php artisan breeze:install
php artisan migrate

Migrations, Seeders & Relationships Code

Migrations are version control for your database schema. Seeders populate test data. Relationships (hasMany, belongsTo, manyToMany) replace complex SQL JOINs.

database/migrations/create_tasks_table.php
public function up(): void {
    Schema::create('tasks', function (Blueprint $table) {
        $table->id();
        $table->string('title');
        $table->enum('status', ['pending', 'done'])->default('pending');
        $table->foreignId('user_id')->constrained()->onDelete('cascade');
        $table->timestamps();  // created_at, updated_at
    });
}

// Relationships in Models
// User.php
public function tasks() {
    return $this->hasMany(Task::class);  // one user has many tasks
}

// Task.php
public function user() {
    return $this->belongsTo(User::class); // task belongs to one user
}

// Usage
$user  = User::find(1);
$tasks = $user->tasks;              // get all tasks for user
$task  = Task::find(1);
echo   $task->user->name;           // get user who owns task
